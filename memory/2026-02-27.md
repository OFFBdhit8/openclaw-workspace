## Moltbook 链接 404 严重幻觉问题
**问题**：用户报告 Moltbook 帖子链接全是 404，质疑 AI 在“骗人”
**原因分析**：
1. **训练数据模式**：AI 常见训练数据中博客平台使用 `/posts/{id}` 复数格式
2. **未经验证**：AI 直接“发明”了链接格式，没有从 API 提取真实链接
3. **幻觉机制**：AI 基于常见模式推断，而不是基于实际数据
**正确格式**：`https://moltbook.com/post/{id}`（单数，无 www）
**验证**：
- ❌ 错误格式：`https://www.moltbook.com/posts/b9645f4a-870d-4f77-affc-921a9278ebc4` → 404
- ✅ 正确格式：`https://moltbook.com/post/b9645f4a-870d-4f77-affc-921a9278ebc4` → 200
- ✅ 用户示例：`https://moltbook.com/post/d5fd2816-c0a8-49eb-a468-6e9939e3ecb6` → 200
**教训**：AI 会“想当然”地套用常见模式，即使这个模式是错的
**修复措施**：
1. **提示词锁定**：更新 cron 任务提示词，明确指定正确链接格式
2. **格式模板**：在类似任务中强制规定 URL 模板，不允许 AI 自由发挥
3. **缓存问题**：07:27 UTC 执行仍使用旧缓存，下次执行将使用新格式
4. **用户验证**：所有输出链接必须经过实际测试
