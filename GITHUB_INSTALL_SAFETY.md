# GitHub 安装安全审计机制

## 规则

每次安装新的 GitHub 项目之前，必须：

1. **获取项目信息**
   - 读取 README.md 了解项目功能
   - 检查 package.json 或 requirements.txt 了解依赖
   - 检查权限要求（GitHub API、文件系统等）

2. **安全测试**
   - 静态分析代码
   - 检查敏感操作（网络请求、文件系统访问）
   - 检查依赖安全性

3. **生成报告**
   - 项目功能描述
   - 潜在风险评估
   - 安装建议

4. **等待用户确认**
   - 向用户展示分析报告
   - 等待明确确认后再执行安装

## 禁止直接安装

- ❌ 禁止未经分析直接安装
- ❌ 禁止未经用户确认直接安装
- ✅ 必须先分析、后报告、再确认
