## 🔄 超额后切换方案

### 方案 A: 手动切换（简单）

当 Doubao 额度用完，手动修改 Cron 任务模型：

```bash
# 编辑 cron 任务配置
openclaw cron edit a8a6e978-bddf-4b88-8382-b5952f05ef3a

# 将模型改为 deepseek-chat
```

### 方案 B: 配置自动 Fallback（推荐）

OpenClaw 目前不支持自动 fallback，但可以：

1. **监控 API 错误** - 定期检查 cron 运行日志
2. **设置告警** - 错误时通知你
3. **快速切换** - 发现错误立即修改配置

### 方案 C: 混合使用（最优）

```
心跳检查 → Doubao Mini (免费额度内)
Cron 任务 → DeepSeek (一直用，不依赖免费额度)
主对话 → Qwen3.5-Plus (独立配额)
```

这样即使 Doubao 免费额度用完，只影响心跳检查（成本很低）。

## 🚨 常见问题

### Q: 免费额度用完会自动扣费吗？

**A**: 不会！Doubao 是后付费模式，免费额度用完后：
- 如果没有充值：API 返回错误，无法调用
- 如果已充值：自动从余额扣费

### Q: 如何避免意外扣费？

**A**: 
1. 不绑定信用卡/支付方式
2. 定期检查用量
3. 设置用量告警（火山引擎控制台）

### Q: Doubao 额度用完后切换哪个模型？

**A**: 推荐切换到：
- `deepseek-chat` - 性价比高，中文好
- `doubao-seed-2.0-lite` - 如果有其他免费额度
- `qwen/qwen-turbo` - 备选方案

### Q: 心跳检查能不能完全关闭？

**A**: 可以！编辑 `HEARTBEAT.md`，清空所有内容即可。但建议保留基本检查。
