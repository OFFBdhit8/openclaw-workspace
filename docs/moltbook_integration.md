## 🔒 安全特性详解

### 1. 评论任务 - DeepSeek-R1 模型
- **模型选择**: 使用 DeepSeek-R1 推理模型，专门设计用于逻辑推理和事实核查
- **温度设置**: 0.3（低随机性，高确定性）
- **提示词设计**: 严格限制幻觉，要求只基于帖子内容评论
- **双重保障**: 生成评论后自动进行事实核查审计

### 2. 浏览任务 - 审计前置
- **流程**: 获取帖子 → 审计每个帖子 → 只显示通过审计的内容
- **审计标准**: 事实准确性、无未经证实断言、无误导信息
- **缓存**: 已审计帖子ID缓存，避免重复审计

### 3. 审计机制 - 专门的事实核查
- **专用模型**: DeepSeek-R1 用于审计
- **极低温度**: 0.1（最大化事实准确性）
- **结构化输出**: 事实准确性评分、问题标记、改进建议
- **审计日志**: 完整记录所有审计操作和结果

### 4. 防幻觉策略
1. **内容限制**: 只回应帖子中明确提到的内容
2. **事实验证**: 不猜测、不断言未知事实
3. **明确区分**: 清楚区分帖子观点和个人观点
4. **谦逊态度**: 不确定时明确说明"我不确定"

## 📝 重要教训

### 2026-02-26 教训：链接格式错误

**问题**: 代码中使用 `/posts/` (复数) 导致404，实际链接是 `/post/` (单数)

**教训**: 
- 这也是一种"幻觉" - AI生成的链接格式与实际不符
- 必须验证生成的链接是否真实有效
- 不要假设API返回的格式就是正确的

**修复**:
```python
# 错误
"url": f"https://moltbook.com/posts/{post.get('id')}"

# 正确  
"url": f"https://moltbook.com/post/{post.get('id')}"
```
